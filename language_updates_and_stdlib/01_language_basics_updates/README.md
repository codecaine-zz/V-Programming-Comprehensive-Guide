# Language basics updates

This section gathers the newer language capabilities that expand on the core V concepts taught earlier in the guide. These examples are intentionally small and practical so you can focus on the idea behind each feature before applying it in a larger project.

## Grouped topics

- [01_core_language_features](01_core_language_features) — options/results, generics, and interfaces for writing safer and more reusable code.
- [02_language_extensions](02_language_extensions) — sum types, attributes, inline assembly, and directives for more advanced and expressive programs.

## Beginner-focused guidance

- Use options and results when a value may be absent or an operation may fail.
- Use generics when you want one function or struct to work with different types without repeating code.
- Use interfaces when several unrelated types should share the same behavior.

A good way to study this section is to read an example, run it, and then ask: “What problem does this solve?” That habit helps beginners move from memorizing syntax to understanding real programming patterns.
